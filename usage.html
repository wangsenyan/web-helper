网页小助手 — 使用文档

一、安装与首次启用

  1. 从 Chrome Web Store 安装网页小助手。审核通过前，商店链接会显示为“审核中”。
  2. 打开你要使用的网站，例如公司后台、CRM、ERP、客服工作台或帮助中心。
  3. 点击 Chrome 右上角扩展图标，选择启用当前网站。
  4. Chrome 弹出权限确认时，点允许。扩展只会在你授权的网站工作。
  5. 页面刷新后，左侧会出现网页小助手入口；配置模型前不会显示字段“?”提示。

二、配置模型

  1. 打开左侧网页小助手，点击设置按钮。
  2. 进入“模型”页，选择供应商并填写模型、Base URL 和 API Key。
  3. 点击“测试连接”。连接通过后，该模型配置才会生效。
  4. 可以添加多个模型，并选择一个默认模型。
API Key 只保存在本地浏览器。自定义模型 endpoint 会先请求对应 API origin 权限，未授权时不会调用。

三、启用和管理网站

  1. 在扩展弹窗中启用当前网站，或在侧栏设置的“网站”页添加当前域名。
  2. 可以停用或删除已添加的网站。
  3. 未授权的网站不会注册内容脚本，也不会出现字段提示、侧栏或右键菜单。
网页小助手安装时不会请求站点权限。启用站点和自定义模型 endpoint 时会按实际域名请求；截图功能因 Chrome captureVisibleTab 要求，会在用户主动截图时请求可选 <all_urls> 权限。不支持 chrome://、扩展商店、浏览器内部页面等受限页面。

四、使用字段智能提示

模型通过且当前站点已启用后，普通输入框、文本域、下拉框和部分组件外壳旁会出现橙色“?”。

  • 点击“?”后，扩展会读取字段标签、占位符、附近 DOM 摘要、页面标题、局部正文和知识库片段。
  • 如果模型返回候选值，可一键填入字段。
  • 密码、验证码、Token、邮箱、手机号、证件号、银行卡等敏感字段不会出现“?”。

五、使用左侧 AI 助手

  • 点击左侧入口展开对话面板。
  • 可以直接询问当前页面、选中文本、字段含义、知识库内容。
  • 选中文本后使用右键菜单“问 网页小助手”，会把选中文本和附近上下文带入对话。
  • 对话记录保存在 chrome.storage.session，浏览器关闭后自动清除。

六、截图提问

  • 支持可视页截图、区域截图和滚动长截图。
  • 截图会作为附件显示在输入框旁，发送前可以预览、删除、复制或另存为。
  • 如果默认模型未标记为支持图片理解，网页小助手不会发送图片本体，而是使用截图区域文字和 DOM 上下文。

七、本地知识库

学习当前页面

在左侧面板的“页面知识”中，可以重新学习当前页面或深度采集页面。登录态页面采集会先提示确认。

导入本地文档

  1. 打开侧栏设置的“知识库”页。
  2. 填写知识库名称和关联站点 URL。
  3. 选择文件。支持 txt、md、html、json、csv、docx、pptx、pdf。
  4. 点击导入文档。文档会在浏览器本地解析并写入本地知识库。

管理知识库

可以预览知识库正文、编辑名称和关联网站、启用/停用或删除知识库。列表会展示页面数、有效字符、原始字符、图片数量和质量标签等统计。

八、隐私与数据

  • 扩展没有开发者自有服务器。
  • 配置、模型 API Key、知识库、缓存和任务状态默认保存在 Chrome 本地存储。
  • 只有当用户主动向 AI 提问、点击字段提示、确认发送截图或采集页面时，相关上下文才会被处理。
  • 发送到模型和写入知识库前会尽量脱敏，但用户仍应避免采集或发送高度敏感信息。

九、常见问题

为什么没有“?”按钮？

  1. 确认当前站点已授权。
  2. 确认默认模型已测试通过。
  3. 刷新页面。
  4. 确认字段不是密码、验证码、邮箱、手机号、Token 等敏感字段。

截图为什么没有作为图片发送？

只有当默认模型配置标记为支持图片理解时，截图图片本体才会发给模型。否则会改用截图区域的文字和 DOM 上下文。

卸载后数据怎么处理？

卸载扩展后，Chrome 会删除扩展保存的本地数据。
